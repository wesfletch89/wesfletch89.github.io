A little bit about me

Hello, my name is Wes Fletcher. I am a Computer Science senior at Washington State University — Tri-cities. I've lived in Washington my whole life but will be moving to Colorado after I finish school in May of 2018. I've been married for 8 years to my high school sweetheart and together we have a 4 year old son.

A childhood friend of mine and I dreamed of making a video game after countless weekends of staying up late playing the latest and greatest. One day in high school, we finally decided to start designing one. Of course, we quickly discovered that we had no idea what we were doing. The graphical style, main characters, and overall storyline came together pretty quickly, but the code...OH, THE CODE! Neither of us had ever written a single line of code. Thus began learning C++ with the beginner's standard program, "Hello World". A future Software Engineer was born. Sadly, we never did finish making that game, but the dream remains.

Fun fact: I originally went to school for Civil Engineering.

I've been fortunate to have worked with some great people throughout my time at WSU. My path in Computer Science has led me to focus on C/C++ and Java development. I've done numerous Desktop and Terminal-based applications (examples found in "Projects"), but I'm most intrigued with Cybersecurity. It is incredibly rewarding to solve complex problems and have the results materialize on screen. I hope that feeling never goes away.

Another fun fact: I taught myself HTML5, CSS3, and a little JQuery, and built this website.

C/C++ Projects

Find Dups

Find Dups takes any number of files as input and recursively searches the passed-in path for all duplicate files.

Example: If given the file "foo.c" and the directory "bar", this program would search the directory "bar" and all its subdirectories for any duplicates of "foo.c".

Who Can

Who Can takes an action that can be performed on a file or directory and a specific file or directory as input and returns a list of all users that have permission to perform that action on that file or directory. The path to the file or directory can be relative or absolute.

Example: If given the action "delete" for the file "~/Desktop/foo.c", the program would return a list of any users who have permission to delete the file "~/Desktop/foo.c".

Snapshot

Snapshot is a function that can be placed inside a body of code to produce a core dump file without stopping the program. In essence, it provides a "snapshot" of the state of the program at the line in which it was called. This can prove especially useful for debugging.

Tattle

Tattle returns a list of users' login/logoff times according to user-defined options. The options that can be given include: specific users, date, and time. If the date is specified but not the time, it defaults to the current time. If the time is specified but not the date, it defaults to the current date. The program also returns users that are still logged in.

Turing Machine

Turing Machine simulates a Turing machine by allowing the user to load a Turing machine definition file, load input strings, and trace operations on an input string to determine if a string is accepted or rejected by the defined language. The program provides the user with a number of commands, such as: listing input strings, showing the current definition, adding/removing input strings, and setting the number of transitions to perform.

Adaptive Huffman

Adaptive Huffman utilizes a user-defined alphabet to encode/decode a message on-the-fly using the Huffman encoding algorithm. It uses an AVL tree to store the most frequently used characters towards the root of the tree, which minimizes encoding size.
